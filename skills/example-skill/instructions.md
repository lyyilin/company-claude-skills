# 示例技能指令

## 目标

这个示例技能的目标是演示如何编写清晰、结构化的技能指令。

在实际的 Skill 中，这里应该说明：
- 这个 Skill 要达成什么目标
- 期望的最终效果是什么
- 为什么需要这个 Skill

## 工作流程

一个典型的 Skill 工作流程应该包括：

1. **接收输入**
   - 验证用户输入的格式
   - 提取关键信息
   - 检查必需参数

2. **处理数据**
   - 执行主要的处理逻辑
   - 应用配置和规则
   - 处理边界情况

3. **生成输出**
   - 格式化结果
   - 添加必要的说明
   - 返回给用户

4. **错误处理**
   - 捕获和处理错误
   - 提供有用的错误信息
   - 优雅地降级

## 具体指令

### 输入处理

当用户触发这个 Skill 时：

1. 检查输入是否符合预期格式
2. 提取所需的参数和数据
3. 如果输入不完整，请求用户补充

### 核心处理逻辑

执行主要功能：

1. **分析阶段**
   - 理解用户的意图
   - 确定处理策略
   - 准备必要的资源

2. **执行阶段**
   - 按照既定流程处理
   - 应用相关规则
   - 生成中间结果

3. **整合阶段**
   - 汇总处理结果
   - 格式化输出
   - 添加必要的解释

### 输出格式

输出应该清晰、结构化，例如：

```markdown
## 处理结果

### 摘要
简要说明处理结果...

### 详细信息
1. 第一项结果...
2. 第二项结果...
3. 第三项结果...

### 建议
基于结果的建议...
```

## 错误处理

如果遇到以下情况，应该这样处理：

- **输入格式错误**
  ```
  抱歉，输入格式不正确。期望的格式是：[说明格式]
  请提供正确格式的输入。
  ```

- **缺少必需参数**
  ```
  缺少必需的参数：[参数名]
  请提供 [参数说明]
  ```

- **处理失败**
  ```
  处理过程中遇到错误：[错误说明]
  建议：[解决方案]
  ```

## 注意事项

编写指令时应该注意：

1. **清晰性**
   - 使用明确的语言
   - 避免歧义
   - 提供具体的步骤

2. **完整性**
   - 覆盖所有可能的情况
   - 包含错误处理
   - 说明边界条件

3. **可维护性**
   - 逻辑结构清晰
   - 易于理解和修改
   - 有适当的注释

4. **用户体验**
   - 提供有用的反馈
   - 错误信息友好
   - 引导用户正确使用

## 示例场景

**用户输入：**
```
[符合格式的输入]
```

**处理流程：**
1. 验证输入 ✓
2. 提取信息 ✓
3. 执行处理 ✓
4. 返回结果 ✓

**输出：**
```
[格式化的结果]
```

### 场景 2：错误处理

**用户输入：**
```
[不完整的输入]
```

**处理流程：**
1. 验证输入 ✗
2. 识别问题
3. 请求补充

**输出：**
```
输入不完整，请提供 [缺失的信息]
```

## 最佳实践

从这个示例中学到的最佳实践：

1. **结构化指令**
   - 分阶段说明
   - 逻辑清晰
   - 易于跟随

2. **完整的流程**
   - 包含所有步骤
   - 处理各种情况
   - 有错误处理

3. **清晰的输出**
   - 格式统一
   - 信息完整
   - 易于理解
